# 🎯 حل مشكلة التعديلات - موقع شفرا

## ✅ المشكلة وحلها

### المشكلة الأصلية:
- البيانات كانت تُحفظ في `localStorage` (ذاكرة المتصفح فقط)
- كل متصفح/جهاز لديه نسخته الخاصة من البيانات
- التعديلات لا تظهر للآخرين لأن كل شخص له بيانات منفصلة

### ✨ الحل:
تم إنشاء **خادم Express** يحفظ جميع البيانات في ملف JSON مركزي (`site-data.json`)
- جميع المتصفحات تستقبل نفس البيانات من الخادم
- عند حفظ تعديل، يتم حفظه في الخادم مباشرة
- جميع المستخدمين يرون التحديثات فوراً

## 🚀 خطوات التثبيت والتشغيل

### 1️⃣ تثبيت المكتبات المطلوبة:
```bash
npm install
```

### 2️⃣ تشغيل الخادم:
```bash
npm start
```

أو للتطوير (مع إعادة تحميل تلقائي):
```bash
npm run dev
```

### 3️⃣ الدخول للموقع:
- الموقع الرئيسي: `http://localhost:3000/index.html`
- لوحة التحكم: `http://localhost:3000/admin.html`

## 📁 الملفات الجديدة:

| الملف | الوصف |
|------|-------|
| `backend-server.js` | خادم Express الرئيسي |
| `api-client.js` | مكتبة للتواصل مع الخادم |
| `admin-updated.js` | لوحة التحكم المحدثة |
| `script-updated.js` | الموقع الرئيسي المحدث |
| `package.json` | المكتبات المطلوبة |
| `site-data.json` | قاعدة البيانات (تُنشأ تلقائياً) |

## 🔄 خطوات الترقية:

### للموقع الحالي (index.html):
1. استبدل `script.js` بـ `script-updated.js` أو أضف هذا السطر في الـ HTML:
```html
<script src="api-client.js"></script>
<script src="script-updated.js"></script>
```

### لوحة التحكم (admin.html):
1. استبدل `admin.js` بـ `admin-updated.js` أو أضف هذا السطر في الـ HTML:
```html
<script src="api-client.js"></script>
<script src="admin-updated.js"></script>
```
